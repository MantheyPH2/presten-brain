## Issues / Concerns

None rising to the level of a flag. Minor notes for awareness:

1. **SnapSoccer game volume estimate** is wide (10k–20k). Not a blocker, but should be tightened when the scraper runs for the first time. Log event count and game count at that point.
2. **Source Submission Framework** leaves email notification as Phase 2 with no timeline. This should be noted in SENTINEL's queue — tournament directors who submit events will expect some feedback. Phase 2 timeline should be set once MVP ships.
3. **Daily Pipeline** does not address what happens if the 2 AM cron overlaps a previous run that hung. The health check addresses detection but not prevention. FORGE should note this as a known gap — a `flock` lock or PID file check on the cron script would prevent overlap. Not a blocker but should be in the implementation notes.
