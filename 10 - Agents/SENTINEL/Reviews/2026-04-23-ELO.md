## What's Good

### Rank Bands Design
The choice of Approach B (rating-value bands) over USA Rank's rank-position bands is the correct call, and the reasoning is sound — cross-age-group consistency matters. A team rating of 1500 meaning "Gold" in U13G and U17B is better than "Gold" meaning different things based on how many teams are enrolled in each age group. The thresholds (Gold ≥1500, down to Green <900) are grounded in Glicko v7 calibration values rather than arbitrary numbers — that's the right foundation.

Using a SQL view (`rankings_with_bands`) rather than a stored column to avoid migration cost during threshold tuning is a smart MVP decision. The 7-month inactivity match with USA Rank creates a consistent user mental model.

**Strength:** Anticipating that thresholds may need adjustment after Presten runs the distribution query. ELO didn't over-commit to numbers that haven't been validated against real data.

### Club Rankings Design
The hybrid identity approach (GotSport org_id primary, name matching fallback) is correct. The 70–80% GotSport coverage estimate is realistic. The decision to produce separate boys/girls rankings is the right call — combined rankings would obscure the many gender-specific clubs in youth soccer.

The 6-step SQL CTE chain for computation is the right architecture — it's auditable, testable step-by-step, and Presten can debug individual CTEs. The 4-week bootstrap plan (Week 1 GotSport seed, Week 2 name fallback) is realistic.

**Strength:** The watch-out note about confirming `gotsport_org_id` column presence before starting is valuable. This is the kind of dependency check that prevents a 2-hour dead-end.

### Event Strength Ratings Design
All 4 required metrics are fully defined with specific numeric thresholds — this is exactly the bar set in the task. The 50-point minimum rating gap for upset detection is a well-reasoned noise filter. The decision to use current ratings as historical approximation (with transparent labeling) is pragmatic and correctly deferred to Phase 2 when `ratings_history` exists.

The "display-only for Phase 1" recommendation on ranking engine integration is correct. The circular dependency argument (feedback loop → full Elo recompute) is sound reasoning. Phase 2 K-factor modifier evaluation is the right path.

**Critical Strength:** The competitive differentiation framing — "They show labels. We show the math." This is the right product philosophy and should carry through to the UI implementation. Percentile, StdDev, upset % as actual numbers is a meaningful advantage over USA Rank's opaque labels.

## Issues / Concerns

### Rank Bands — Missing Pre-Flight Validation Step
The design correctly notes Presten must run the distribution query before implementing. However, the design doc doesn't specify what Presten does if the median differs significantly from ~1050. ELO's implementation plan task (assigned below) must include a threshold recalibration protocol. This is a dependency that needs to be resolved before the view is created, not after.

**Resolution:** The implementation plan task includes a contingency formula for threshold adjustment.

### Club Rankings — 22-Hour Estimate Accuracy
The 22-hour estimate spanning 4 weeks is credible but means this won't be demo-ready at DSS (May 16). ELO's briefing correctly flags this — Club Rankings are a post-DSS feature. SENTINEL agrees. The GotSport org_id dependency check must happen in Week 1 or the entire timeline slips. Presten needs to run `SELECT club, gotsport_org_id FROM teams LIMIT 20` before any other club ranking work begins.

**No new task assigned for this** — the dependency check is a 5-minute Presten action, not a FORGE/ELO task. Flagging it here for Presten's awareness.

### Cross-Cutting: `last_game_date` Column Verification
All three designs reference `last_game_date` in the `rankings` table. ELO's briefing correctly flagged this as unverified. This needs to be confirmed before any of the three features are implemented — if the column doesn't exist, the inactivity logic in all three systems fails silently or errors.

**Resolution:** Added to ranking accuracy audit task as a pre-flight check. Also flagged in this review for Presten's immediate awareness.
