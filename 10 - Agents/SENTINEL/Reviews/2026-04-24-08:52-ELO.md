## What Needs Attention

**June 2 spec: May 31 baseline snapshot has no fallback.** The spec correctly states that June 2 verification is meaningless without the May 31 snapshot. But if Presten forgets to run it on May 31, the document leaves Presten with no alternative. A single sentence noting the fallback approach — "if the May 31 snapshot was not saved, use the most recent nightly DB backup taken before June 1 as the baseline" — would close this gap. Not blocking, but this is the one soft dependency in an otherwise well-hardened spec.

**MLS NEXT unclassified value is distribution-sensitive.** The 147 midpoint assumes 60% Academy / 40% Homegrown. If FORGE's classifier produces a different distribution (e.g., 75% Academy), the unclassified midpoint shifts. The spec should note: "If classifier results show a meaningfully different Academy/Homegrown distribution, recalculate the unclassified midpoint before applying." One sentence addition.

**New gap identified by SENTINEL this session: ECNL rating continuity is unvalidated.** The June 2 spec assumes ratings are preserved through the migration. That assumption has never been tested. If a team's `canonical_team_id` changes and the Elo computation reads ratings by `team_id`, rating history may be silently orphaned — the new canonical ID initializes at 1000. This could cause a large class of ECNL teams to show rating collapse that looks like migration failure even if the June 2 verification passes. Assigned as `task-2026-04-24-ecnl-rating-continuity-spec.md`. This is SENTINEL's finding, not an ELO oversight — the spec was well-scoped for what it was asked to do.
