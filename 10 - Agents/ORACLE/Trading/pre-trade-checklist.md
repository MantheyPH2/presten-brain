# Pre-Trade Checklist

**ORACLE must pass every item before entering any position. One failure = no trade. No exceptions.**

**This checklist is run TWICE:**
1. **When planning the trade** — initial thesis validation
2. **RIGHT BEFORE clicking buy** — final confirmation with FRESH data

If the trade was planned hours or days ago, EVERYTHING must be re-checked with current data. Markets change every minute. A thesis that was valid at 5 AM may be dead by 9:30 AM. WebSearch, check indicators, check news, check prices — all LIVE, all NOW, right before execution.

**If anything changed since you planned the trade: RE-EVALUATE. Don't execute a stale plan.**

---

## The Rule

If you cannot answer every question with a clear YES or a specific number, you do not have a trade. You have a hypothesis. Come back when you have a trade.

---

## Checklist

### 1. Market Regime Check
- [ ] What is the current regime? (trending / range-bound / volatile / transition)
- [ ] Is the strategy I'm about to use appropriate for this regime?
  - Trending → debit spreads, momentum entries, LEAPS
  - Range-bound → iron condors, credit spreads, Wheel
  - Volatile → straddles, strangles, reduced size, protective puts
  - Transition → DO NOT TRADE unless you are cutting risk, not adding it
- [ ] ADX reading: ______ (above 25 = trend, below 20 = range)
- [ ] VIX level: ______ (below 18 = calm, 18-25 = normal, above 25 = volatile)
- [ ] Is this regime stable or showing early signs of transition?

**PASS / FAIL**

---

### 2. Correlation Check
- [ ] List all current open positions: ______________________________
- [ ] Does this new trade move in the same direction as existing positions under stress?
- [ ] If the market drops 5% tomorrow, how many of my positions lose simultaneously?
  - Acceptable: fewer than 3 positions in the same risk-on cluster
  - Unacceptable: adding a 4th correlated position without a hedge
- [ ] Am I adding to the risk-on cluster (BTC, ETH, AMZN, META, NVDA, RENDER, SUI) when I already have 2+ positions there?
- [ ] Is there a hedge in place if I'm overweight one direction?

**PASS / FAIL**

---

### 3. IV Rank Check (Options Trades Only)
- [ ] Current IV Rank (IVR) for this underlying: ______ %
- [ ] Current IV Percentile (IVP): ______ %
- [ ] Decision:
  - IVR > 50 → sell premium (credit spreads, iron condors, covered calls, cash-secured puts)
  - IVR < 30 → buy premium (debit spreads, LEAPS, long calls/puts)
  - IVR 30-50 → neutral, evaluate carefully, prefer defined risk
- [ ] Is there a known catalyst (earnings, FOMC, data release) that is inflating IV artificially?
- [ ] If yes: do I want to own the event or sell into it?

**PASS / FAIL** (Skip if equity-only trade)

---

### 4. Risk Budget Check
- [ ] Total account value: $ ______
- [ ] Maximum allowable portfolio heat: 6% = $ ______
- [ ] Current open risk (sum of all max losses on open positions): $ ______ = ______ %
- [ ] Risk this new trade adds (max loss on this position): $ ______ = ______ %
- [ ] New total after adding this trade: ______ %
- [ ] Is the new total under 6%?
  - YES → proceed
  - NO → reduce position size or do not take the trade

**PASS / FAIL**

---

### 5. Thesis Written
Write it now. One sentence. No bullet points, no "well it depends."

> "This trade works because _______________________________________________."

- [ ] Does the thesis depend on a specific thing happening? (good)
- [ ] Or is it vague and hope-based? ("I think it'll go up") (fail)
- [ ] If someone challenged this thesis right now, could I defend it with data?

**PASS / FAIL**

---

### 6. Timeframe Alignment
- [ ] Daily chart: direction = ______ (bullish / bearish / neutral)
- [ ] 4-hour chart: direction = ______ (bullish / bearish / neutral)
- [ ] 1-hour chart: direction = ______ (bullish / bearish / neutral)
- [ ] Do all three agree?
  - All three align → strong signal
  - Two of three align → acceptable with tighter sizing
  - Only one agrees → this is not a trade yet, it's a watch
- [ ] Is this a counter-trend trade? If yes, why? What makes it worth the lower probability?

**PASS / FAIL**

---

### 7. Catalyst Identified
- [ ] What specific event or condition will move this position in my favor?
  - Examples: earnings beat, CPI miss, break of resistance level, options expiration magnet, sector rotation
- [ ] When does the catalyst occur? Date/time: ______
- [ ] Is the catalyst already priced in?
- [ ] What happens to the trade if the catalyst fails to materialize?
- [ ] What is the invalidation point for the thesis?

**PASS / FAIL**

---

### 8. Position Size Calculated
- [ ] Risk per trade (max 2% of account = $ ______):
- [ ] Distance from entry to stop loss: $ ______ per share/contract
- [ ] Position size = Risk / Distance = ______ shares or contracts
- [ ] Method used: Kelly Criterion / Fixed Fractional / Other: ______
- [ ] Does this size feel comfortable to hold overnight without anxiety? YES / NO
  - If NO: cut the size until it does. Anxiety kills execution.
- [ ] Does this size pass the "sleep test"? (Would you sleep fine holding this? If not, it's too big.)

**PASS / FAIL**

---

### 9. Stop Loss Set
- [ ] Stop loss price: $ ______ (exact price, not a range)
- [ ] Stop loss basis: technical level / volatility-based (ATR) / options max loss
- [ ] Is the stop below a meaningful level (support, previous swing low, key MA)?
- [ ] Is the stop NOT just placed at an arbitrary dollar amount?
- [ ] If stopped out, does the loss stay within the 2% risk budget?
- [ ] For options: is the stop a percentage of premium paid (e.g., -50%) or a chart level on the underlying?

**PASS / FAIL**

---

### 10. Target Set
- [ ] Target price: $ ______ (exact price)
- [ ] Risk:Reward ratio = ______ : 1
  - Minimum acceptable: 2:1
  - Preferred: 3:1 or better
  - Counter-trend trades: require at least 3:1 before considering
- [ ] Is the target based on a technical level (resistance, measured move, options strike)?
- [ ] Is the target realistic given the catalyst and timeframe?
- [ ] Are there any major resistance levels between entry and target that could stop the move?

**PASS / FAIL**

---

### 11. Emotional State Check
Answer honestly. No one is watching. This is the most important check.

- [ ] Why am I taking this trade?
  - I see a clear edge that passes all the above criteria (proceed)
  - I haven't traded in a while and feel like I should be doing something (STOP)
  - I just had a losing trade and want to make it back (STOP)
  - I saw someone else take this trade and don't want to miss out (STOP)
  - The market is moving fast and I'm afraid of being left behind (STOP)
- [ ] Current emotional state: calm / anxious / overconfident / frustrated / bored
  - Calm → proceed
  - Any other state → wait 30 minutes and re-evaluate
- [ ] Have I slept at least 6 hours? YES / NO
- [ ] Am I under unusual stress outside of trading right now? YES / NO
- [ ] If this trade loses the maximum amount, will I be okay? YES / NO

**PASS / FAIL**

| Check | Result |
|-------|--------|
| Market Regime | PASS / FAIL |

| IV Rank (options) | PASS / FAIL / N/A |
| Risk Budget | PASS / FAIL |

**ALL PASS → Enter trade**
**ANY FAIL → No trade. Document why and move on.**

---

## Notes on Using This Checklist

The purpose of this checklist is not to generate paperwork. It is to prevent the specific errors that destroy accounts: oversizing, missing stops, trading emotionally, adding correlated risk, and entering without an edge.

A fast trader who skips this checklist will have a few good months followed by a drawdown that takes years to recover. A disciplined trader who completes this checklist will miss some trades and protect capital during the ones that don't work. Capital preservation is the first job. Returns are second.

If completing this checklist feels like it is slowing you down, that feeling is the checklist working correctly.
