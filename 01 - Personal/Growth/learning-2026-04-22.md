## What to Learn Today

### Primary: Multi-Agent Orchestration Patterns

**Why it matters:** As your vault and business agents multiply, the naive approach (each agent runs independently) will hit coordination failures — duplicate work, conflicting recommendations, no shared memory. The solution is orchestration patterns: how agents hand off tasks, share context, and escalate decisions.

**Concepts to grasp:**
- **Supervisor pattern** — one orchestrator agent routes tasks to specialist sub-agents (this is roughly what you have now, with Claude Code as the implicit orchestrator)
- **Peer-to-peer pattern** — agents communicate directly via shared state or message queues; better for real-time but harder to debug
- **Blackboard pattern** — agents all read/write to a shared "blackboard" (your Obsidian vault is functionally a primitive blackboard already)
- **ReAct loops** — agents that reason, act, observe, and repeat; the cognitive engine behind most production agent systems

**Time investment:** 45–60 minutes

**Where to learn:**
- Anthropic's agent documentation (claude.ai/docs) — they now have a full section on multi-agent design
- "Building Effective Agents" blog post from Anthropic (late 2025) — still the clearest articulation of when to use agents vs. not
- LangGraph conceptual docs (even if you never use LangGraph, the graph-based mental model for agent flows is valuable)

### Secondary: Claude's Extended Thinking for Analysis Tasks

**Why it matters:** Claude's extended thinking mode produces meaningfully better analysis on complex reasoning tasks — market analysis, business model evaluation, strategic planning. If you ever wire ORACLE to run autonomously with API access, enabling extended thinking on the market scan and idea generation steps would substantially improve output quality.

**Time investment:** 20 minutes

**Action:** Read the `extended-thinking` section of the Anthropic API docs. Note the token implications — extended thinking uses significantly more tokens, so it's worth being intentional about when to enable it.

## Skill to Practice This Week

**Prompt chaining by hand** — before automating a multi-step workflow, run it manually once: write the output of step 1, feed it as input to step 2, observe where the chain breaks down. This will make you dramatically better at designing agent pipelines.

Try it with: the ORACLE market scan workflow. Write a "raw signals" note, then write the "analysis" note from it, then write the "recommendation" note. Notice which step loses information.
